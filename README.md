# Сбор данных российских ИТ-компаний

Проект выполнен в рамках тестового задания: сбор базы российских ИТ-компаний со штатом от 100 сотрудников и выше.

## Структура проекта:

    | Файл | Назначение |
    |------|------------|
    | `main.py` | Основной скрипт парсера |
    | `requirements.txt` | Зависимости Python |
    | `README.md` | Документация проекта |
    | `.gitignore` | Игнорируемые файлы |
    | `*.html` (8 файлов) | HTML-страницы с Rusprofile |
    | `companies.csv` | Результат работы парсера |

## Установка зависимостей
    pip install -r requirements.txt

## Запуск парсера
    python main.py

## Источники данных
Сайт Rusprofile.ru - открытая база данных российских компаний.

## Метод сбора

### Ручная фильтрация на Rusprofile:
    Фильтр "Количество сотрудников: от 100"
    Поиск ИТ-компаний по фильтру ОКВЭД
    Сохранение заранее отфильтрованных HTML-страниц по федеральным округам
### Программная обработка:
    Парсинг HTML-файлов с помощью BeautifulSoup
    Извлечение структурированных данных
    Экспорт в CSV

## Что удалось реализовать:
    Собрано 296 компаний
    Автоматический парсинг 8 региональных файлов
    Экспорт в структурированный CSV
    Код запускается одной командой
    Файл companies.csv содержит данные компаний со следующими полями:
## Cтруктура данных в csv файле
    inn - ИНН компании
    name - Название компании
    region - Регион расположения
    okved_main - Основной ОКВЭД
    reg_date - Дата регистрации
    revenue - Выручка за год
    revenue_year - Год выручки
    source - Источник данных

## Что можно было бы улучшить:
    Автоматизация сбора через Selenium
    Валидация ОКВЭД на соответствие ИТ-сфере
    Добавление контактных данных (сайт, email)

## Использование LLM
В проекте использовались языковые модели (QWEN, DeepSeek) только для:
    Понимания структуры HTML-страниц Rusprofile
    Подсказок по CSS-селекторам для парсинга
    Исправления ошибок, с которыми в процессе отладки возникали наибольшие проблемы.

### Примечания
    HTML-файлы включены в репозиторий для обеспечения повторяемости
    Данные актуальны на момент сохранения страниц
    Код предназначен для тестового задания
    Весь код доступен в репозитории и может быть запущен командой python main.py.
